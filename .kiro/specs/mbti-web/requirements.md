# 要件定義書

## プロジェクト概要
MBTI Bot Webアプリ化 - Next.js + React で既存CLIロジックをWebアプリに変換。診断機能、タイプ/機能/軸の閲覧、Claude APIによるAI要約をブラウザから利用可能にする。既存のドメインロジック（builtin-data.js, diagnosis-engine.js, url-mapper.js, wiki-fetcher.js, claude-api.js, template-renderer.js）を再利用する。

## 要件一覧

### REQ-1: ホームページ（メインメニュー）
**説明:** アプリのエントリポイント。各機能（診断、タイプ一覧、認知機能一覧、軸一覧）へのナビゲーションを提供する。
**受入基準:**
- メインメニューに4つの機能へのリンクが表示される
- レスポンシブレイアウトで表示される
- 日本語UIで表示される

### REQ-2: 診断ページ
**説明:** 16問の質問に回答し、MBTI診断結果を表示するページ。
**受入基準:**
- `getQuestions()` から取得した16問が順次表示される
- 各質問に対して選択肢（A/B）を選べる
- 全問回答後に `POST /api/diagnosis` を呼び出し結果を取得する
- 診断結果としてタイプ名、スコア内訳が表示される
- 結果画面からタイプ詳細ページへ遷移できる

### REQ-3: タイプブラウザページ
**説明:** 16タイプの一覧表示と、選択したタイプのカテゴリ別詳細表示。
**受入基準:**
- `getAllTypes()` から取得した全16タイプが一覧表示される
- タイプ選択後、カテゴリ（概要・認知機能・性格特性等）を選択できる
- 選択したカテゴリのWikipedia情報 + AI要約が表示される
- ローディング状態が適切に表示される

### REQ-4: 認知機能ブラウザページ
**説明:** 8つの認知機能（Se, Si, Ne, Ni, Te, Ti, Fe, Fi）の一覧と詳細表示。
**受入基準:**
- 全認知機能が一覧表示される
- 選択した機能のWikipedia情報 + AI要約が表示される

### REQ-5: 軸ブラウザページ
**説明:** 4つの軸（E/I, S/N, T/F, J/P）の一覧と詳細表示。
**受入基準:**
- 全軸が一覧表示される
- 選択した軸のWikipedia情報 + AI要約が表示される

### REQ-6: API Route - POST /api/diagnosis
**説明:** 診断回答を受け取り、`diagnosis-engine.js` の `evaluate()` を呼び出して結果を返すエンドポイント。
**受入基準:**
- リクエストボディ: `{ answers: string[] }`（16要素の配列、各要素は "A" or "B"）
- レスポンス: `{ type, scoreBreakdown, scores }`
- バリデーションエラー時は400を返す

### REQ-7: API Route - GET /api/content
**説明:** Wikipedia記事を取得しClaude APIで要約するエンドポイント。
**受入基準:**
- クエリパラメータで対象（type/function/axis）とキーを指定
- サーバーサイドで `wiki-fetcher.js` と `claude-api.js` を使用
- `ANTHROPIC_API_KEY` はサーバーサイドのみで使用し、クライアントに露出しない
- レスポンス: `{ summary, rawText, heading }`

### REQ-8: API Route - GET /api/types
**説明:** 全タイプ・認知機能・軸のマスタデータを返すエンドポイント。
**受入基準:**
- `builtin-data.js` から取得した全データを返す
- レスポンス: `{ types, functions, axes }`

### REQ-9: レスポンシブデザイン
**説明:** Tailwind CSSを使用したモバイルファースト・レスポンシブデザイン。
**受入基準:**
- モバイル（< 640px）、タブレット（640-1024px）、デスクトップ（> 1024px）で適切に表示
- タッチフレンドリーなUI要素

### REQ-10: ダーク/ライトモード
**説明:** ユーザーの好みに応じたテーマ切り替え機能。
**受入基準:**
- システム設定に連動したデフォルトテーマ
- ヘッダーにトグルボタンを配置
- 選択はlocalStorageに保存

### REQ-11: セキュリティ
**説明:** APIキーの保護とセキュアなAPI設計。
**受入基準:**
- `ANTHROPIC_API_KEY` は `.env.local` に保存しサーバーサイドのみで参照
- クライアントバンドルにAPIキーが含まれない
- API Routeで適切な入力バリデーションを実施
